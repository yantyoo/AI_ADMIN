# XpERP AI 챗봇 관리자 기능정의서

- 기준일: 2026-04-03
- 범위: 현재 프론트 구현 기준 최신본
- 원칙: 삭제된 기능은 제외하고, 현재 화면에 남아 있는 기능만 정리한다.

## 1. 공통 정책

| 항목 | 정책 | 비고 |
|---|---|---|
| 인증 흐름 | 아이디/비밀번호 로그인 후 OTP 확인을 거쳐 대시보드로 이동 | `test0000`, `test1111`은 mock 계정 |
| 세션 유지 | 로그인 성공 시 세션 저장, 아이디 저장은 선택값만 localStorage에 보관 | 비밀번호 저장 금지 |
| 권한 기준 | `MASTER`, `OPERATOR` 2종만 사용 | `USER`는 관리자 접근 불가 |
| 메뉴 노출 | 현재 로그인 프로필의 `role` 기준으로 사이드바 메뉴를 필터링 | MASTER는 전체, OPERATOR는 대시보드/피드백/지식 기반 조회 노출 |
| 공통 레이아웃 | 좌측 사이드바 + 상단 헤더 + 본문 콘텐츠 영역 | 본문은 화면 높이를 채우는 구조 |
| 리스트 공통 패턴 | 리스트 카드 내부를 `상단(타이틀/액션) - 중단(테이블/스크롤) - 하단(페이지네이션)`으로 구성 | `ListPanel` 패턴 사용 |
| 리스트 스크롤 | 데이터가 많을 때 전체 페이지가 아니라 리스트 영역 내부에서만 스크롤 | `overflow-y: auto` |
| 페이지네이션 | 목록형 화면은 10개 단위 페이지네이션 적용 | 이전/다음, 현재 페이지 active, 생략부호 지원 |
| 빈 상태 | 데이터가 없거나 검색 결과가 없으면 안내 문구 노출 | 카드 높이는 유지 |
| 버튼 규격 | 공통 버튼 높이와 정렬을 통일 | `primary / secondary / danger / icon` 공통 베이스 사용 |
| 오류 상태 | 로딩/빈 상태/오류 상태를 화면 단위로 노출 | 현재 구현 우선 |

## 4. 운영 정책

| 항목 | 정책 |
|---|---|
| 접근 제한 | 메뉴는 role 기준으로 숨기고, 직접 진입은 인증된 세션만 허용한다 |
| MASTER 범위 | 콘텐츠, 캐시 Q&A, 지식 기반 조회, 계정/권한 관리 포함 전체 운영 가능 |
| OPERATOR 범위 | 대시보드, 피드백 관리, 지식 기반 조회 사용 가능 |
| 목록 UX | 리스트는 화면 하단까지 채우고, 본문 영역만 스크롤한다 |
| 데이터가 적을 때 | 빈 공간은 리스트 카드의 확장 영역으로 자연스럽게 흡수한다 |
| 데이터가 많을 때 | 테이블 내부 스크롤과 페이지네이션을 함께 사용한다 |
| 삭제/상태 변경 | 모든 파괴적 동작은 확인 모달을 거친다 |
| 최신화 원칙 | 이 문서는 현재 프론트 구현 기준 최신본만 유지한다. 변경 이력은 별도 관리하지 않는다 |

## 5. 팝업/버튼 상세 규격

### 5.1 팝업 공통 규격
| 항목 | 규칙 |
|---|---|
| 오픈 방식 | 리스트 행 선택, 버튼 클릭, 상태 변경, 삭제, 등록, 수정 동작에서 모달을 연다 |
| 닫기 방식 | 우측 상단 닫기, `취소`, backdrop 클릭, `ESC` 입력으로 닫을 수 있다 |
| 예외 | 파괴적 동작 확인 모달과 저장 진행 중 모달은 backdrop 클릭으로 닫히지 않도록 한다 |
| 기본 버튼 순서 | 왼쪽 `취소`, 오른쪽 `확인/저장/삭제` 순서를 유지한다 |
| 중복 요청 | `확인` 클릭 후에는 응답 완료 전까지 버튼을 `loading/disabled` 상태로 전환한다 |
| 입력 검증 | 필수 입력값이 충족되기 전에는 `확인/저장/등록` 버튼을 비활성화한다 |
| 메시지 | 성공/실패/중복/권한 없음 메시지는 팝업 내부 또는 토스트로 즉시 노출한다 |
| 포커스 | 팝업 열림 시 첫 입력값 또는 첫 버튼에 포커스를 이동한다 |

### 5.2 로그인/OTP 팝업 상세
| 팝업 | 버튼 |
|---|---|
| 로그인 | `취소` / `로그인` |
| OTP 인증 | `취소` / `확인` / `재전송` |
| 검증 | 아이디, 비밀번호, OTP 값이 유효해야 활성화된다 |
| 실패 | 인증 실패 시 팝업은 유지하고 오류 문구만 갱신한다 |
| 재전송 | OTP 재전송은 제한 시간과 횟수 제한을 둘 수 있다 |

### 5.6 버튼 공통 규격
| 항목 | 규칙 |

## 6. 캐시 Q&A 상세 규칙

### 6.1 입력 규격

| 항목 | 최소 | 최대 | 기본값 | 화면 표기 |
|---|---:|---:|---|---|
| 질문 | 1자 | 500자 | 없음 | `현재/500자` |
| 답변 | 1자 | 2000자 | 없음 | `현재/2000자` |
| 검색어 | 2자 권장 | 제한 없음 | 없음 | `2자 이상 입력 권장` |
| 상태 | 1개 | 1개 | ACTIVE | 셀렉트 |

### 6.2 목록/상세 규칙

| 항목 | 규칙 |
|---|---|
| 목록 컬럼 | 질문 / 답변 요약 / 상태 / 등록일 / 수정일 |
| 상세 패널 | 질문 박스 / 답변 박스 / 상태 뱃지 / 수정자 / 수정일 / 조회수 |
| 박스 높이 | 질문/답변 박스는 동일한 최소 높이로 표시 |
| 빈 상태 | 선택 항목 없음과 검색 결과 없음 문구를 분리 표시 |
| 검색 힌트 | `2자 이상 입력하면 검색 정확도가 올라갑니다.` 노출 |

### 6.3 모달/팝업 규칙

| 모달 | 필드 | 버튼 | 검증 |
|---|---|---|---|
| Q&A 등록 | 질문, 답변, 상태 | `초기화` / `등록` | 질문/답변 필수, 중복 질문 차단 |
| Q&A 수정 | 질문, 답변, 상태 | `초기화` / `수정 저장` | 질문/답변 필수, 중복 질문 차단 |
| Q&A 삭제 확인 | 삭제 대상 확인 문구 | `취소` / `삭제` | 선택 항목 없으면 동작 불가 |
| 상태 전환 | 상세 패널 상태 버튼 | 즉시 적용 | 확인 팝업 없음 |

### 6.4 상태/예외 규칙

| 항목 | 정책 |
|---|---|
| 활성 | 캐시 응답에 포함 |
| 비활성 | 목록에는 노출하되 캐시 응답에는 미포함 |
| 중복 판정 | 소문자화, 앞뒤 공백 제거, 연속 공백 제거, 특수문자 제거 후 비교 |
| 필수값 누락 | 저장 버튼 비활성 또는 인라인 오류 노출 |
| 검색 결과 0건 | `조건에 맞는 Q&A가 없습니다.` 표시 |
| 상세 미선택 | `Q&A를 선택하면 상세 정보가 표시됩니다.` 표시 |

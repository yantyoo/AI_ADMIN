# XpERP AI 관리자 IA 상세서

- 기준일: 2026-04-03
- 목적: 기능정의서의 FS 항목을 실제 화면 구조로 연결해, 개발과 QA가 바로 사용할 수 있는 IA 문서로 정리한다.
- 원칙: 화면명만 적지 않고, `뎁스 / 연결 기능 / 사용자 메시지 / 상태 / 예외`까지 함께 적는다.

## 1. IA 해석 기준

| 구분 | 정의 | 예시 |
|---|---|---|
| L1 | 메뉴 단위 화면 | `/feedback`, `/accounts` |
| L2 | 화면 내부 주요 영역 | 리스트, 상세, 요약 카드, 검색 바 |
| L3 | 인터랙션 단위 | 버튼, 입력, 필터, 모달, 토스트 |
| 메시지 | 사용자에게 즉시 보여주는 문구 | 성공, 실패, 빈 상태, 권한 없음 |

## 2. 전역 화면 정책

| 항목 | 정책 |
|---|---|
| 접근 제어 | 메뉴 노출은 role 기준, 직접 진입은 인증 세션 기준으로 제어한다 |
| 레이아웃 | 좌측 사이드바, 상단 헤더, 본문 영역으로 구성한다 |
| 목록 패턴 | 리스트형 화면은 `ListPanel`을 사용하며, 카드가 화면 전체를 억지로 채우지 않도록 한다 |
| 스크롤 원칙 | 페이지 전체가 아니라 리스트 본문 또는 상세 본문만 스크롤한다 |
| 빈 상태 | 데이터가 없으면 카드 상단에 붙는 형태로 안내 문구를 노출한다 |
| 모달 | 저장/삭제/상태 변경은 확인 모달 또는 등록 모달을 사용한다 |
| 공통 메시지 | 성공/실패/권한 없음/빈 결과는 화면 내 인라인 메시지로 즉시 전달한다 |

## 3. 메뉴 구조와 뎁스

| 순서 | 메뉴 | 뎁스 | 접근 권한 | 연결 기능 |
|---|---|---|---|---|
| 1 | 대시보드 | L1 | MASTER, OPERATOR | FS-011~016 |
| 2 | 콘텐츠 관리 | L1 | MASTER | FS-017~028 |
| 3 | 캐시 Q&A 관리 | L1 | MASTER | FS-029~041 |
| 4 | 지식 기반 조회 | L1 | MASTER, OPERATOR | FS-042~049 |
| 5 | 피드백 관리 | L1 | MASTER, OPERATOR | FS-050~056 |
| 6 | 계정/권한 관리 | L1 | MASTER | FS-057~067 |

## 4. 공통 메시지 표준

| 상태 | 표준 메시지 |
|---|---|
| loading | `데이터를 불러오는 중입니다.` |
| empty | `조건에 맞는 데이터가 없습니다.` |
| error | `데이터를 불러오지 못했습니다. 다시 시도해 주세요.` |
| forbidden | `접근 권한이 없습니다.` |
| success | `정상 처리되었습니다.` |
| duplicate | `이미 등록된 항목입니다.` |

## 5. 화면별 IA 상세

### 5.1 로그인

| 항목 | 내용 |
|---|---|
| 뎁스 | L1 |
| 연결 기능 | FS-001~010 |
| 주요 영역 | 로그인 폼, OTP 모달, 상태 메시지 |

| UX 포인트 | 로그인 실패 시 폼은 유지하고, 메시지만 갱신한다 |
| 예외 | 세션이 이미 인증 상태면 `/dashboard`로 이동한다 |

## 7. QA 체크 기준

| 체크 항목 | 합격 기준 |
|---|---|
| 뎁스 표기 | 각 화면이 L1 기준으로 정리되어 있고, 내부 인터랙션은 L3로 분리되어 있어야 한다 |
| 기능 연결 | 모든 주요 화면이 FS-번호와 연결되어 있어야 한다 |
| 메시지 | 성공/빈/오류/권한 없음 메시지가 최소 1개 이상 명시되어야 한다 |
| 상세 정렬 | 피드백 상세는 대화 내용이 위, 부정 사유가 아래에 있어야 한다 |
| 리스트 높이 | 데이터가 적을 때 카드가 화면을 억지로 채우지 않아야 한다 |

## 8. 변경 원칙

| 항목 | 원칙 |
|---|---|
| 기능 추가 | 새 화면이나 모달이 생기면 FS 번호와 IA 섹션을 함께 추가한다 |
| 메시지 변경 | 메시지는 UI와 문서가 동시에 바뀌어야 한다 |
| 구조 변경 | 높이/스크롤/정렬 변경은 공통 레이아웃 규칙에 먼저 반영한다 |
| 문서 우선순위 | 실제 구현과 다르면 문서가 아니라 구현을 기준으로 맞춘다 |
## 9. 캐시 Q&A 화면 상세

### 9.1 L2 구성

| 영역 | 구성 |
|---|---|
| 좌측 | Q&A 목록, 검색, 상태 필터, 페이지네이션 |
| 우측 | 질문 상세, 답변 상세, 상태, 수정 정보, 조회수, 액션 버튼 |
| 팝업 | 등록 모달, 수정 모달, 삭제 확인 모달 |

### 9.2 L3 구성

| L3 | 요소 | 규격 / 메시지 |
|---|---|---|
| 검색 | 질문 검색 입력 + 검색 버튼 | `2자 이상 입력 권장` |
| 상태 필터 | 전체 / 활성 / 비활성 | 셀렉트 |
| 목록 | 질문 / 답변 요약 / 상태 / 등록일 / 수정일 | 10개 단위 페이지네이션 |
| 상세 질문 | 질문 본문 박스 | 최소 높이 동일 적용 |
| 상세 답변 | 답변 본문 박스 | 최소 높이 동일 적용 |
| 상세 메타 | 수정자 / 수정일 / 조회수 | 정보 라벨 고정 |
| 액션 | 수정 / 상태 전환 / 삭제 | 버튼 |
| 빈 상태 | 상세 미선택 | `Q&A를 선택하면 상세 정보가 표시됩니다.` |

### 9.3 팝업 규격

| 팝업 | 필드 | 버튼 | 검증 / 메시지 |
|---|---|---|---|
| 등록 모달 | 질문(500자), 답변(2000자), 상태 | `초기화` / `등록` | 필수값 누락 시 오류 노출 |
| 수정 모달 | 질문(500자), 답변(2000자), 상태 | `초기화` / `수정 저장` | 중복 질문 차단 |
| 삭제 확인 | 삭제 대상 확인 문구 | `취소` / `삭제` | 삭제 후 캐시 응답 제외 안내 |

### 9.4 화면 문구

| 위치 | 문구 |
|---|---|
| 검색 힌트 | `2자 이상 입력하면 검색 정확도가 올라갑니다.` |
| 질문 입력 카운트 | `현재/500자` |
| 답변 입력 카운트 | `현재/2000자` |
| 비어 있음 | `조건에 맞는 Q&A가 없습니다.` |
